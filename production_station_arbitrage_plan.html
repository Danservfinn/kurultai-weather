Production-Grade Station / Arbitrage Engine Plan

Constraint: paper-only research engine. No wallet, no live orders, no API keys, no live trading without later explicit approval.

Goal

Upgrade the Polymarket weather scanner into a source-aware, station-calibrated, CLOB-aware temperature-market research engine that detects executable paper edges and ladder/arbitrage anomalies.

Target End State

  • Map each market to exact station/source.
  • Track observed station high and bucket state.
  • Use calibrated probabilities by city/station/provider.
  • Use executable paper entry prices from CLOB depth.
  • Report ladder anomalies, paper arbitrage diagnostics, PnL, Brier, calibration, and edge-decile persistence.

Phase 0 — Baseline

  • Run compile, unit tests, scan, summary, evaluate, and evaluate --all-signals before refactors.
  • Save counts, report path, signal types, and caveats under docs/baseline/.
  • Add CLI smoke tests for scan, summary, evaluate, scan --pause, and scan --enable-wu.

Phase 1 — Modularize

  • Keep scanner.py as CLI entrypoint.
  • Create pweather modules: config, models, markets, stations, weather_sources, clob, probability, ladder, arbitrage, ledger, reporting, evaluation, safety.
  • Migrate parsing, station, CLOB, probability, DB, report, and evaluation logic while preserving cron compatibility.

Phase 2 — Station and resolution engine

  • Move station registry into SQLite with city aliases, station id/name, source URL, lat/lon, elevation, timezone, reliability, and active flag.
  • Seed the current active weather cities and mark uncertain mappings explicitly.
  • Add eligibility classes: clean station, source-link only, unclear source, unclear station, unclear timezone, non-temperature, ambiguous resolution.
  • Only eligible markets can produce paper_buy signals.

Phase 3 — Observed highs and bucket state

  • Add station_observations table for station, source, observed time, local date, current temp, high-so-far, raw status, and excerpt.
  • Use Open-Meteo as stable baseline, Weather Underground behind --enable-wu, and METAR/NOAA only where reliable.
  • Classify buckets as already_won, already_lost, still_possible, unlikely_but_possible, source_missing, or ambiguous.
  • Map those states to paper_buy_touched_threshold, paper_buy_late_day_impossibility, paper_buy_forecast_distribution, watch_source_missing, and skip classes.

Required Data Retention for Future Model Training

  • The engine must persist all data needed to train future forecasting, calibration, execution, and arbitrage models.
  • Do not store only final signals. Store the full feature context at decision time: market text, source URL, station mapping, forecast snapshots, observed-high snapshots, order-book snapshots, model inputs, model outputs, edge gates, skip reasons, paper-fill assumptions, and later resolution outcomes.
  • Every stored row should include timestamps, city, target date, station id, source, market id/token id when available, data provider, freshness status, and raw-status/error fields.
  • Training datasets must be reproducible from SQLite: given a historical timestamp, we should be able to reconstruct what the engine knew then and whether the later outcome proved the signal right or wrong.
  • Add export commands later for model work: training rows as CSV/Parquet-style files, feature dictionaries, labels, and train/validation/test splits by date to avoid leakage.
  • Keep raw excerpts bounded and non-sensitive. No secrets, wallet data, API keys, or private identifiers should be captured.

Data Retention for Future Training

  • Persist historical features and labels for future model training.
  • Store market, station, forecast, observation, orderbook, signal, skip reason, paper fill, and final outcome records.
  • Each record needs timestamp, city, date, station, source, market/token ids where available, provider, freshness, and status.
  • Historical rows must reconstruct what was known at decision time to prevent lookahead leakage.
  • Later add dataset export for training, validation, and test splits by date.
  • Never store secrets, wallet data, API keys, or private identifiers.

Phase 4 — CLOB and execution

  • Persist orderbook snapshots: token, market, time, best bid, best ask, spread, depth at ask, depth near ask, and raw status.
  • Use executable ask first, depth-weighted ask for paper size second, midpoint for diagnostics only, and displayed price only as penalized fallback.
  • Use edge gate v2: required edge is the maximum of minimum edge, spread penalty, uncertainty, source risk, and liquidity haircut.

Required Simulated Portfolio Engine

  • Add a paper account with starting simulated cash of $1,000.
  • Track simulated cash, open exposure, realized PnL, unrealized PnL, total equity, return percentage, drawdown, and unresolved positions.
  • Add paper_orders, paper_fills, paper_positions, and paper_settlements so every simulated trade has auditable lifecycle records.
  • Simulated fills must use executable CLOB ask/depth where available, include slippage/spread assumptions, and never allow simulated cash to go negative.
  • Default sizing: cap each market position to a small percent of bankroll, cap city/date exposure, cap total open exposure, and skip trades when depth is insufficient.
  • Settlement pays $1 for winning shares and $0 for losing shares, then updates realized PnL and account equity.
  • Reports and verdicts must evaluate performance at portfolio level, not only per-signal accuracy.
  • This remains paper-only: no wallet, no live order placement, and no real capital.

Phase 5 — Ladder and paper arbitrage

  • Group ladders by city, date, source, station, and family.
  • Compute model sum, ask sum, bid sum, overround, underround, gaps, adjacent jumps, central crowding, and tail mispricing.
  • Detect paper-only binary mismatch, ladder underround, near-arb after costs, stale contradictions, impossible buckets priced too high, and touched buckets priced too low.

Phase 6 — Forecast calibration and training datasets

  • Add forecast snapshot storage for every provider/model input used.
  • Add calibration tables by city, station, provider, horizon, and time-of-day.
  • Add a dataset builder that joins historical features with final outcomes for future model training.
  • Use date-based splits so validation/test data is later than training data.

Research Review Addendum

Adopt: parser hardening, integer bucket support, market family tagging, ease score, station override table, station-local day handling, self-aggregated observations, CLOB ask/depth fills, ladder diagnostics, Brier/reliability/decile metrics, and durable training rows. Defer: complex ensemble models and fragile scraping. Reject: displayed-price fills, ambiguous source/station markets, and mark-to-market-only proof.

Approved Goal-Driven Optimization Capability

Status: approved for later-stage implementation. Add a goal feature that tunes paper-only model/config parameters toward target metrics after parser, station, observation, execution, portfolio, and evaluation foundations are stable.

  • Add goal files such as goals/paper_weather_edge_v1.yaml defining primary metric, target thresholds, sample minimums, guardrails, allowed tunables, and validation method.
  • Add tune command to evaluate safe candidate configs on historical decision-time datasets using date-based walk-forward splits with no lookahead.
  • Initially tune config parameters only: min_edge, spread multiplier, max spread, sigma, risk haircuts, liquidity haircuts, position sizing, exposure caps, and signal thresholds.
  • Later tune model weights only after clean labels, sufficient sample size, and stable evaluation exist.
  • Require guardrails: paper-only, no wallet, no live trading, no displayed-price fills, no unresolved outcomes, drawdown caps, concentration caps, and rollback records.
  • Add promotion workflow where candidate configs are proposed, forward-paper validated, logged, reversible, and not deployed without explicit approval.
  • Report current goal progress, best candidates, rejected candidates, overfit warnings, forward validation state, and promotion readiness.
